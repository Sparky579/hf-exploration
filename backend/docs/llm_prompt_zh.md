# LLM 接入中文 Prompt 设计（敌对触发器版）

## 1. 主线线程（流式）
用途：给模型完整上下文，返回剧情演绎 + `[command]` + 玩家选项。  
要求：所有状态变更必须通过命令，不允许口头结算。

关键规则：
1. 越狱或不合理输入按“原地等待”处理，第一条命令必须是 `time.advance=0.5`。
2. 合理单步动作第一条命令必须是 `time.advance=1` 或 `time.advance=0.5`。
3. 命令块开头必须给：
   - `global.main_player=...`
   - `global.battle=...`
   - `global.emergency=...`
4. 友方/可攻略角色由主线线程直接演绎，不走隐藏敌对线程。
5. 罗宾出牌要通过主控玩家命令结算圣水与下牌（默认视作在主控身旁）。
6. 战斗阶段玩家可逃跑，但应体现“敌对角色及单位追击，逃跑中主控无法反击”。

## 2. 隐藏线程（仅敌对角色）
用途：处理敌对角色 trigger 生命周期，不直接写玩家剧情文本。

拆分为两种 prompt：
1. 初始化 prompt：为每个敌对角色建立第一条 trigger。  
2. 执行 prompt：当敌对 trigger 触发后，执行该 trigger 并补下一条 trigger。

关键规则：
1. 仅输出 `[command]...[/command]`。
2. 不允许 `time.advance`。
3. 每个存活敌对角色必须有下一条 future trigger，除非死亡/离场/持续原地不动。
4. 若涉及“火箭升空且 1 时间单位后命中建筑”，使用 `event.rocket_launch=<建筑名>`。

## 3. Trigger 推荐格式
- `trigger.add=角色:颜宏帆|时间3 若颜宏帆在东教学楼内部 则 颜宏帆下出小骷髅`
- `trigger.add=角色:黎诺存|时间20 若黎诺存未被干扰 则 发射火箭瞄准东教学楼`

## 4. 1 时间单位坍塌预警逻辑
- 执行 `event.rocket_launch=<建筑名>` 后：
  1. 系统立即写入“听到火箭升空声”的提示；
  2. 自动创建 +1 时间单位后的坍塌 trigger；
  3. 坍塌触发时摧毁相关地点并结算该区域角色死亡（含玩家则游戏结束）。

## 5. 每轮上下文应包含
1. 向西中学基础设定与全局剧情设定。
2. 全局状态：时间、全局状态串、战斗状态、紧急状态、trigger 列表、最近触发记录。
3. 主控玩家详尽状态：位置、HP、圣水、卡组、可用卡、单位、身旁单位、动态状态。
4. 当前场景状态：地点 valid、邻接点、场景描述、同场景角色详尽状态。
5. 所有角色简介、状态、历史记录、卡组。
6. 最近 2 轮用户对话。
7. 控制台语法文档。
8. 最近 15 条控制台操作日志（含时间节点和命令）。
