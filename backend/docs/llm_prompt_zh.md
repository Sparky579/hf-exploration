# LLM 接入中文 Prompt 设计

## 主线叙事请求（流式）
用途：给模型完整上下文，返回剧情演绎 + `[command]` 命令块 + 玩家选项。  
要求：所有状态变更必须通过命令，不允许“口头结算”。

核心约束：
1. 用户输入不合理或越狱时，视为原地等待，命令中先 `time.advance=0.5`。
2. 合理单步动作，命令中先 `time.advance=1` 或 `0.5`。
3. 命令块第一行必须明确全局主控状态：
   - `global.main_player=...`
   - `global.battle=...`
   - `global.emergency=...`
4. 已入队/已离场角色不重复发现或邀请。
5. 输出格式固定：
   - `【剧情】`
   - `[command] ... [/command]`
   - `【选项】`（至少 3 项）

## 旁角色懒更新请求（并行）
用途：只在玩家到达关联地点时，补全关联角色在时间窗口内的行为轨迹。

核心约束：
1. 仅输出 `[command] ... [/command]`，不输出剧情。
2. 不使用队列命令（不使用 move/deploy 入队），直接状态强改。
3. 只更新关联角色，其他角色不输出。
4. 保持人设：
   - 罗宾温和、反应慢。
   - 马超鹏机敏。
   - 许琪琪、冬雨存在感较强且不可 OOC。

## 上下文字段（每轮都传）
1. 向西中学基础设定与全局剧情设定。
2. 全局状态：时间、全局状态串、战斗状态、紧急状态、trigger 列表、最近触发记录。
3. 主控玩家详尽状态：位置、HP、圣水、卡组、可用卡、单位、身旁单位、动态状态。
4. 当前场景状态：地点 valid、邻接点、场景描述、同场景角色详尽状态。
5. 所有角色简介、状态、历史记录、卡组。
6. 最近 2 轮用户对话。
7. 控制台语法文档。
8. 最近 15 条控制台操作日志（含时间节点和命令）。
